## 1. 專案概述

### 1.1 要解決的問題
小學班級學生常常不自覺就會愈來愈吵鬧。老師口頭提醒的效果短暫、且會消耗老師情緒能量。需要一個**外在的、非人形的制約機制**，讓學生學會**自我監控與自制**。

### 1.2 解決方案概念
做一個網頁版的「班級電子寵物」，透過麥克風偵測教室音量，讓寵物依音量變化狀態，並長期累積成長。**目標是讓學生看得到自己集體行為的即時後果**，把抽象的「安靜」轉換成具體的「保護寵物、幫寵物成長」。

### 1.3 服務對象
- **主使用者**：臺南市安佃國小（及其他有意採用的學校）1-6 年級任課老師
- **間接使用者**：各班學生（主要觀看者，不直接操作）
- **專案擁有者**：賢哥負責設計、維護、推廣

### 1.4 設計哲學（重要）

> **「偵測不到的音量，也就表示音量是可接受的安靜。」**
> — 賢哥，設計定案階段

這句話定義了整個系統的寬容度：
- **我們不是噪音監測儀**，是**提醒自制的陪伴者**
- 寧可偵測不到一些噪音，也不要誤判一個咳嗽就懲罰全班
- 不製造罪惡感、不扣分到負、不讓寵物「死亡」
- **自然後果** > 懲罰：寵物會「躲起來」，而不是「被你們害死」

## 2. 使用場景

### 2.1 典型使用情境
1. 老師進教室，打開筆電投影 → 網頁已在書籤，點開 → 寵物出現
2. 授課開始，老師按「🤫 專心模式」→ 系統開始偵測
3. 學生安靜 → 寵物開心跳舞、音量條綠色、心情值上升
4. 學生開始聊天 → 寵物躲起來、音量條紅色
5. 安靜下來 → 寵物探頭、恢復跳舞
6. 下課前累積足夠點數 → 升階動畫 + 扭蛋抽卡
7. 老師按 🎭 表演鍵 → 寵物展示抽到的新技能，當下課獎勵

### 2.2 特殊情境
- **學生發表、討論活動**：老師切「💬 討論模式」或 ⏸ 暫停偵測
- **考試、嚴格安靜時段**：老師切「🤫 專心模式」+ 🔕 靜音通知
- **月考、段考**：🔕 靜音模式（關彈窗、關音效，但寵物仍偵測累積）
- **下課、自由活動**：切「🎨 自由模式」或 ⏸ 暫停偵測

## 3. 核心機制

### 3.1 雙層機制（短期心情 + 長期技能）

本系統採「混合型」生命週期，同時具備：

- **短期心情值**：每節課重置，立即反饋當下行為
- **長期經驗值 / 技能樹**：跨日累積，永久保留，驅動成長

這兩層機制並行運作，彼此影響：心情值不直接轉換為經驗值，但「安靜秒數」同時累積到這兩個池子。

### 3.2 心情值系統

#### 3.2.1 基本規則
- 範圍：0–100
- 每節課開始時：從 **60** 起跳（中性偏正向）
- 每秒更新一次，依音量判讀加減

#### 3.2.2 音量判讀與心情增減（專心模式預設值）

| 判讀結果 | 心情值增減/秒 |
|---|---|
| 安靜（< 閾值低標） | +0.3 |
| 可接受（閾值低標 ~ 中標） | 0（不動） |
| 偏吵（閾值中標 ~ 高標） | −0.5 |
| 超吵（> 閾值高標） | −1.5 |

> 其他模式（討論、自由）倍率與閾值不同，見 §3.6。

#### 3.2.3 心情視覺對應與防抖機制（重要）

**核心原則：分離「內部計算」與「視覺呈現」**

內部心情值每秒精確更新（公平累積），但寵物的視覺狀態切換必須經過三層防抖，避免邊界震盪、瞬時尖峰、頻繁閃爍造成的教學干擾。

##### 基本狀態對應

| 狀態 | 視覺呈現 | 配套 |
|---|---|---|
| 😄 開心 | 寵物跳舞、愛心特效、背景晴天 | 可選輕快 BGM |
| 🙂 平靜 | 待機動畫、偶爾眨眼 | 無 |
| 😟 擔心 | 躲到洞口、只露尾巴 | 背景轉陰 |
| 😱 害怕 | 完全躲進山洞、畫面轉暗 | 文字提示「太吵了...小精靈不敢出來」（非斥責用詞） |

##### 防抖機制 ①：滯後閾值（Hysteresis，差 6 分）

進入狀態的門檻與退出狀態的門檻不同，避免卡邊界時反覆切換。

| 狀態 | 進入門檻 | 退出門檻 |
|---|---|---|
| 😄 開心 | 心情 ≥ **83** | 心情 < **77** |
| 🙂 平靜 | 53 ~ 77 | — |
| 😟 擔心 | 心情 ≤ **47** | 心情 ≥ **53** |
| 😱 害怕 | 心情 ≤ **17** | 心情 ≥ **23** |

##### 防抖機制 ②：不對稱連續時間要求

寵物狀態切換不是瞬間的，必須心情值跨過閾值並**連續維持特定秒數**：

| 方向 | 連續秒數 | 設計理由 |
|---|---|---|
| **退步**（開心 → 平靜 → 擔心 → 害怕） | **10 秒** | 給學生補救機會、避免一次咳嗽就懲罰 |
| **恢復**（害怕 → 擔心 → 平靜 → 開心） | **5 秒** | 安靜立即有回饋、符合行為學「獎勵要即時」原則 |

設計哲學：**獎勵要即時，懲罰要延遲**。這是經典行為強化理論，也符合本系統「陪伴而非監控」的定位。

##### 防抖機制 ③：切換漸變動畫

一旦確認要切換狀態，動畫過渡用 **1.5-2 秒漸變**（淡入淡出），不是瞬間 snap 切換。視覺感受舒適、不突兀。

##### 各元素更新頻率總表

| 元素 | 更新頻率 | 原因 |
|---|---|---|
| 音量條（UI） | 100ms 即時 | 老師需要即時音量資訊 |
| 內部心情值 | 每秒更新 | 內部公平計算，不顯示也行 |
| **寵物視覺狀態** | **滯後 6 分 + 連續 10/5 秒 + 漸變 1.5-2 秒** | 穩定、不干擾教學 |
| 背景天氣 | 跟隨寵物狀態同步切換 | 視覺一致性 |

#### 3.2.4 心情恢復機制
- 吵完後只要安靜下來，**寵物會在 5 秒後慢慢探頭回來**
- 給學生「補救機會」，避免一次失誤整節課都挽回不了
- 恢復比退步快，鼓勵學生立即改善行為

### 3.3 成長階段（6 階段）

#### 3.3.1 階段與預設門檻

| 階段 | 代號 | 標準門檻（總經驗值） | 備註 |
|---|---|---|---|
| 🥚 蛋 | egg | 0 | 初始狀態 |
| 🐣 初生 | newborn | 300 | 第一次升階儀式 |
| 🐥 幼年 | young | 800 |  |
| 🦊 成長 | growth | 1800 |  |
| 🦄 進階 | advanced | 3500 |  |
| ✨ 完全體 | final | 6000 | 可延伸「傳說階段」 |

#### 3.3.2 老師可調門檻
老師設定頁提供三套預設 + 完全自訂：
- **寬鬆**（低年級/活潑班級）：門檻為標準 × 0.6
- **標準**：上表數值
- **嚴格**（高年級/安靜班級）：標準 × 1.5
- **自訂**：6 個階段各一個欄位，老師直接填

### 3.4 技能系統

#### 3.4.1 普通技能（N 級，6 個）
每升一階自動解鎖 1 個普通技能。內容舉例：

| 階段 | 技能 | 表演動作 |
|---|---|---|
| 初生 | 轉圈 | 寵物開心轉一圈，灑星星特效 |
| 幼年 | 彩色泡泡 | 吐出彩虹泡泡，點擊可破 |
| 成長 | 魔法帽 | 戴帽子變出隨機小驚喜 |
| 進階 | 小翅膀 | 學會飛，日常動畫升級 |
| 完全體 | 唱歌 | 短 BGM 表演（老師可關） |
| 完全體+ | 發光體 | 發光效果 + 光環 |

#### 3.4.2 稀有技能（抽卡機制）

##### 稀有度分級
| 等級 | 保底機率 | 視覺光效 |
|---|---|---|
| N（普通） | 100%（保底） | 白光 |
| R（稀有） | 30% | 藍光 |
| SR（超稀有） | 8% | 紫光 |
| SSR（傳說） | 2% | 金光 |

##### 三種觸發時機
1. **升階保底扭蛋**：每次升階自動扭一次，至少保底 N，有機率更高
2. **里程碑扭蛋**：每累積 500 點（老師可調）自動扭一次
3. **閃耀時刻扭蛋**：連續 3 節課心情值平均 > 90 觸發；只抽 SR 與 SSR 池

##### 稀有技能範例
- **R 級**：雙胞胎分身、黑板寫字、煙火秀、戴眼鏡變學霸樣
- **SR 級**：節慶皮膚（聖誕、萬聖、春節）、特殊背景（星空、海底、森林）、寵物生日動畫
- **SSR 級**：黃金皮膚、會說話（預錄語音）、會跟小精靈講悄悄話、隱藏進化外型

##### 抽卡動畫
- 3 秒扭蛋機動畫
- 卡片翻面效果
- 依稀有度播對應光效（白/藍/紫/金）
- 卡片存入「相簿」永久保存

### 3.5 音量偵測

#### 3.5.1 校準流程（首次使用）
1. 老師第一次開啟網頁 → 跳「校準精靈」
2. 畫面提示：「請讓教室保持一般安靜 10 秒，小精靈要認識你們的教室」
3. 系統抓取這 10 秒的 RMS 音量作為「環境基線」
4. 基線 + 預設倍率 → 計算出該教室的低/中/高三個閾值
5. 存入 localStorage，以後自動套用

#### 3.5.2 情境模式切換

| 模式 | 圖示 | 低標 | 中標 | 高標 | 適用場合 |
|---|---|---|---|---|---|
| 專心 | 🤫 | 基線 +3dB | 基線 +6dB | 基線 +10dB | 講課、考試、寫作業 |
| 討論 | 💬 | 基線 +8dB | 基線 +12dB | 基線 +18dB | 分組、發表 |
| 自由 | 🎨 | 基線 +15dB | 基線 +22dB | 基線 +30dB | 下課、自由活動 |

> 數值為設計起點，MVP 上線後依老師實測回饋調整。

#### 3.5.3 微調機制
- 每個模式下方提供「+」「−」鍵
- 每按一次微調 ±1dB
- 老師自己抓手感，設定存入該模式
- 「重新校準」按鈕可重置

#### 3.5.4 技術實作
- Web Audio API → getUserMedia → AnalyserNode
- 採樣：AnalyserNode.getFloatTimeDomainData() → 計算 RMS
- 平滑化：3 秒 moving average，避免單次咳嗽觸發
- 更新頻率：心情值每秒更新，UI 音量條每 100ms 更新

### 3.6 音量偵測與心情值的關聯

| 模式 | 判讀 | 心情值/秒 |
|---|---|---|
| 🤫 專心 | 安靜 | +0.3 |
| | 可接受 | 0 |
| | 偏吵 | −0.5 |
| | 超吵 | −1.5 |
| 💬 討論 | 安靜 | +0.3 |
| | 可接受 | 0 |
| | 偏吵 | −0.2 |
| | 超吵 | −0.6 |
| 🎨 自由 | 任何音量 | 0（不累積、不扣減） |
| ⏸ 暫停 | 不偵測 | 不變化 |

## 6. 技術堆疊

### 6.1 前端
- **HTML5 + Vanilla JavaScript**（不用 build tool）
- **Alpine.js**（3KB）：輕量響應式狀態
- **Tailwind CSS**（CDN 版）：快速切版
- **Web Audio API**：音量偵測（瀏覽器內建）

### 6.2 資料
- **localStorage**：主要儲存
- **JSON 匯出**：備份

### 6.3 部署
- **Cloudflare Pages** 或 **GitHub Pages**：免費靜態部署
- 提供固定網址 + QR Code
- **PWA（Progressive Web App）**：加到主畫面像 App、可離線

### 6.4 美術
- **MVP 階段**：用 Gemini Nano Banana（AI 生圖 skill）生 6 個成長階段 + N 級技能 SVG
- **Phase 2**：持續用 AI 擴充稀有技能圖
- **Phase 3 可選**：請美術老師或付費素材升級

### 6.5 音效
- 免費音效庫（freesound.org 或 pixabay）
- 老師可全關

### 6.6 無後端
完全前端運作。零部署成本、零維護、零隱私風險。

**範圍**：

## 9. 未決事項與日後擴充

### 9.1 未決（等 MVP 上線後老師反饋）
- 三種模式的預設閾值（+3/+6/+10 等）是否合理
- 心情值增減速率是否適當（怕太快/太慢）
- 6 階段門檻的「標準」值會不會偏難/偏易
- 抽卡機率是否符合期待

### 9.2 日後可擴充
- **多寵物切換**：老師帶多班時，每班一隻寵物，切換使用
- **班級排行榜**（慎重）：全校老師看得到哪班寵物等級高
- **期末結業儀式**：畢業班的寵物「光榮退役」動畫
- **Self-hosted 雲端存檔**：如果真的需要多裝置同步
- **互動元素**：學生舉手點寵物、觸發隱藏動畫

### 9.3 明確不做
- ❌ 監控個別學生（違反教育倫理、家長疑慮）
- ❌ 上傳音訊到雲端（只分析音量、不錄音）
- ❌ 帳號系統（違反「零管理成本」原則）
- ❌ 付費功能（公益用途，全校免費用）

## 10. 風險與考量

### 10.1 技術風險（賢哥已評估解除）
- ✅ 麥克風權限：學校電腦無限制
- ✅ 瀏覽器：老師用 Chrome
- ✅ 音量敏感度：賢哥另備麥克風；偵測不到 = 可接受的安靜
- ✅ 教室噪音基線：老師各自校準調整

### 10.2 教育倫理考量
- 不監控個別學生，只分析教室集體音量
- 寵物設計避免「死亡」「受傷」等負面意象
- 視覺/文字提示用鼓勵語氣，非斥責
- 老師有完全控制權（重置、暫停、靜音）

### 10.3 推廣與接受度
- 需要一套簡單明瞭的「30 秒上手」教學
- 不同老師風格差異大，**彈性 > 標準化**
- 賢哥會先在 1-2 個班試用，回饋後再推廣

## 附錄 A：設計決策記錄（ADR）

### ADR-001：為何選「一班一隻寵物」而非「每生一隻」
- **脈絡**：考慮過每位學生一隻寵物相互競爭
- **決策**：一班一隻
- **理由**：
  1. 目標是學會「集體自制」，競爭會讓孩子互相指責
  2. 共同擁有感 > 個人競爭感
  3. 老師管理成本低、學生心理壓力小
  4. 符合「同舟共濟」的班級文化

### ADR-002：為何選「環境響應式」而非「獎懲雙向」
- **脈絡**：討論純獎勵、獎懲雙向、環境響應三種哲學
- **決策**：環境響應式（躲/逃，不扣到負分、不死亡）
- **理由**：
  1. 避免「你們害死寵物」的罪惡感
  2. 符合自然後果教育法
  3. 1-6 年級都能接受
  4. 即時視覺反饋（寵物躲 = 現在太吵），不用解釋

### ADR-003：為何選「localStorage」而非雲端
- **脈絡**：討論雲端同步 vs 本機儲存
- **決策**：localStorage + JSON 手動備份
- **理由**：
  1. 教育現場避免上傳學生行為資料
  2. 零後端 = 零部署 = 零維護
  3. 老師 99% 時間用同一台筆電
  4. JSON 匯出夠用於換機

### ADR-004：為何加抽卡機制
- **脈絡**：擔心「只有升階」長期動機不足
- **決策**：加入 N/R/SR/SSR 抽卡系統
- **理由**：
  1. 驚喜感 = 長期動機（學生會期待下次抽到什麼）
  2. 稀有度讓已達成完全體的班級仍有目標
  3. 儀式感（全班一起看抽卡）強化集體記憶
  4. 低成本實作，僅需新增圖素
